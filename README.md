Дипломный проект по профессии «Веб-разработчик»
Дипломный проект представляет собой создание сайта для бронирования онлайн билетов в кинотеатр и разработка информационной системы для администрирования залов, сеансов и предварительного бронирования билетов.
Сущности
Кинозал. Помещение, в котором демонстрируются фильмы. Режим работы определяется расписанием на день. Зал — прямоугольное помещение, состоит из N х M различных зрительских мест.
Зрительское место. Место в кинозале. Есть два вида: VIP и обычное.
Фильм. Информация о фильме заполняется администратором. Фильм связан с сеансом в кинозале.
Сеанс. Временной промежуток, во время которого в кинозале будет показываться фильм. На сеанс могут быть забронированы билеты.
Билет. QR-код c уникальным кодом бронирования, в котором обязательно указаны место, ряд, сеанс. Билет действителен строго на свой сеанс. Для генерации QR-кода можно использовать сервис.
Роли пользователей системы
Администратор — авторизованный пользователь.
Гость — неавторизованный посетитель сайта.
Возможности администратора
Создание или редактирование залов.
Создание или редактирование списка фильмов.
Настройка цен.
Создание или редактирование расписания сеансов фильмов.
Возможности гостя
Просмотр расписания.
Просмотр списка фильмов.
Выбор места в кинозале.
Бронирование билета на конкретную дату.
Важные моменты
Должна присутствовать валидация входных данных на стороне сервера.
Пароль должен храниться в захешированном виде и при аутентификации должна быть проверка хеша пользователя.

Окружение разработки: Windows 10/11, PHP 8.5.1, Laravel 13.x, Composer, SQLite, Blade, HTML/CSS/JavaScript без обязательного npm/Vite.

Получение и подготовка проекта
1.Распакуйте или скопируйте проект в локальную папку.
2.Откройте эту папку в VS Code.
3.Откройте терминал в корне проекта.
4.Убедитесь, что рядом находятся artisan и composer.json.

Установка зависимостей Composer
composer install
Команда устанавливает зависимости из composer.lock. После завершения должна появиться папка vendor.
Проверка:
composer show
composer show endroid/qr-code
Предупреждение. Если composer.lock присутствует, используйте composer install, а не composer update. Команда update может обновить версии пакетов и изменить рабочее окружение проекта.
Если QR-пакет отсутствует, установите его отдельно:
composer require endroid/qr-code

Основные параметры .env для локального запуска:
APP_NAME="ИдёмВКино"
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://127.0.0.1:8000

DB_CONNECTION=sqlite
DB_DATABASE=C:/Projects/cinema_booking/database/database.sqlite

Создание SQLite-базы данных
Создайте пустой файл базы данных:
PowerShell
New-Item database/database.sqlite -ItemType File -Force
CMD
type nul > database\database.sqlite
Проверьте, что файл существует:
Get-Item database/database.sqlite
Важно. Рабочая база проекта хранится в одном файле database/database.sqlite. Перед опасными операциями достаточно сделать копию этого файла.

Миграция базы данных
Выполните все миграции:
php artisan migrate
Проверьте их состояние:
php artisan migrate:status
У всех миграций проекта должен быть статус Ran.

Публичный доступ к постерам
Фильмы хранят постеры в storage/app/public. Создайте публичную ссылку:
php artisan storage:link
После этого Laravel создаст:
public/storage  ->  storage/app/public
Важно. Если постеры не отображаются, проверьте наличие public/storage и значение poster_path в таблице movies.

Создание администратора
При запуске на чистой базе таблица users может быть пустой. Если в проекте нет готового сидера, администратора можно создать через Tinker.
php artisan tinker
use App\Models\User;
use Illuminate\Support\Facades\Hash;

User::create([
    'name' => 'Administrator',
    'email' => 'admin@example.com',
    'password' => Hash::make('ChangeMe123!'),
]);
Для выхода из Tinker:
exit

апуск локального сервера
php artisan serve
