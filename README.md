# 🎬 ИдёмВКино

## Дипломный проект по профессии «Веб-разработчик»

**«ИдёмВКино»** — веб-приложение для просмотра расписания кинотеатра, выбора мест и онлайн-бронирования билетов.

Проект также включает административную информационную систему для управления:

* кинозалами;
* зрительскими местами;
* фильмами;
* расписанием сеансов;
* стоимостью билетов;
* бронированиями;
* электронными билетами с QR-кодом.

## 📖 Описание проекта

Дипломный проект представляет собой сайт для онлайн-бронирования билетов в кинотеатр.

Гость может:

* посмотреть расписание;
* выбрать фильм и сеанс;
* увидеть схему кинозала;
* выбрать свободное место;
* забронировать билет;
* получить электронный билет с QR-кодом.

Администратор может управлять залами, фильмами, расписанием, ценами и продажами билетов.

---

## 🧩 Основные сущности

### Кинозал

Помещение, в котором демонстрируются фильмы.

Кинозал представляет собой прямоугольную схему из `N × M` зрительских мест. Для каждого зала можно настроить:

* название;
* количество рядов;
* количество мест в ряду;
* типы зрительских мест;
* стоимость обычных и VIP-мест;
* доступность продажи билетов.

### Зрительское место

Место в кинозале.

Поддерживаются следующие типы мест:

* обычное;
* VIP;
* недоступное для бронирования.

### Фильм

Информация о фильме создаётся и редактируется администратором.

Для фильма указываются:

* название;
* описание;
* продолжительность;
* страна производства;
* постер.

Фильм может быть связан с одним или несколькими сеансами.

### Сеанс

Временной промежуток, в течение которого в определённом кинозале демонстрируется фильм.

Для сеанса указываются:

* фильм;
* кинозал;
* дата;
* время начала.

На каждый сеанс пользователи могут бронировать билеты.

### Бронирование

Содержит информацию о выбранном пользователем сеансе и зрительском месте.

Система не должна позволять забронировать место, которое уже занято другим пользователем.

### Билет

Электронный билет содержит:

* уникальный код бронирования;
* название фильма;
* дату и время сеанса;
* название кинозала;
* номер ряда;
* номер места;
* стоимость;
* QR-код.

Билет действителен только на указанный в нём сеанс.

## 📚 Установка зависимостей

Выполните в корне проекта:

```bash
composer install
```

Команда установит зависимости, зафиксированные в файле `composer.lock`.

После завершения установки должна появиться папка:

```text
vendor
```

Проверить установленные зависимости:

```bash
composer show
```

Проверить наличие библиотеки для QR-кодов:

```bash
composer show endroid/qr-code
```

> [!IMPORTANT]
> Если в проекте присутствует файл `composer.lock`, используйте команду `composer install`, а не `composer update`.
>
> Команда `composer update` может изменить версии пакетов и привести к несовместимости с текущей версией проекта.

Если пакет для создания QR-кодов отсутствует, установите его отдельно:

```bash
composer require endroid/qr-code
```

## 🔧 Настройка окружения

Создайте файл `.env` из примера:

### PowerShell

```powershell
Copy-Item .env.example .env
```

### CMD

```cmd
copy .env.example .env
```

Основные параметры файла `.env`:

```env
APP_NAME="ИдёмВКино"
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://127.0.0.1:8000

DB_CONNECTION=sqlite
DB_DATABASE=C:/Projects/cinema_booking/database/database.sqlite
```

В параметре `DB_DATABASE` необходимо указать полный путь к файлу базы данных на вашем компьютере.

Пример:

```env
DB_DATABASE=C:/Users/User/Desktop/cinema_booking/database/database.sqlite
```

> [!NOTE]
> В файле `.env` рекомендуется использовать прямые слеши `/`, даже при работе в Windows.

После настройки `.env` сгенерируйте ключ приложения:

```bash
php artisan key:generate
```

Очистите кэш конфигурации:

```bash
php artisan optimize:clear
```

## 🗄 Создание базы данных

Проект использует SQLite. Вся база данных хранится в одном файле:

```text
database/database.sqlite
```

### Создание файла через PowerShell

```powershell
New-Item database/database.sqlite -ItemType File -Force
```

### Создание файла через CMD

```cmd
type nul > database\database.sqlite
```

### Проверка файла через PowerShell

```powershell
Get-Item database/database.sqlite
```

> [!IMPORTANT]
> Перед выполнением опасных операций рекомендуется создать резервную копию файла `database/database.sqlite`.

Например:

```powershell
Copy-Item database/database.sqlite database/database_backup.sqlite
```

## 🔄 Миграции

Для создания таблиц выполните:

```bash
php artisan migrate
```

Проверить состояние миграций:

```bash
php artisan migrate:status
```

У всех выполненных миграций должен отображаться статус:

```text
Ran
```

Для полного пересоздания базы данных можно использовать:

```bash
php artisan migrate:fresh
```

> [!WARNING]
> Команда `php artisan migrate:fresh` удаляет все таблицы и все данные из базы.

Если в проекте используются сидеры:

```bash
php artisan migrate:fresh --seed
```

## 🖼 Публичный доступ к постерам

Постеры фильмов хранятся в каталоге:

```text
storage/app/public
```

Создайте публичную символическую ссылку:

```bash
php artisan storage:link
```

После выполнения команды Laravel создаст связь:

```text
public/storage → storage/app/public
```

Если постеры не отображаются, проверьте:

1. наличие папки или ссылки `public/storage`;
2. наличие файла в `storage/app/public`;
3. значение поля `poster_path` в таблице `movies`;
4. правильность значения `APP_URL` в файле `.env`.

## 👤 Создание администратора

После запуска проекта на чистой базе таблица `users` может быть пустой.

Если готовый сидер отсутствует, администратора можно создать через Laravel Tinker.

Запустите Tinker:

```bash
php artisan tinker
```

Выполните:

```php
use App\Models\User;
use Illuminate\Support\Facades\Hash;

User::create([
    'name' => 'Administrator',
    'email' => 'admin@example.com',
    'password' => Hash::make('ChangeMe123!'),
]);
```

Для выхода из Tinker:

```text
exit
```

Данные для входа:

```text
Email: admin@example.com
Пароль: ChangeMe123!
```

> [!WARNING]
> После первого входа рекомендуется заменить тестовый пароль администратора.

## 🔐 Безопасность

В проекте реализованы основные требования безопасности:

* серверная валидация входных данных;
* защита форм с помощью CSRF-токенов;
* хранение паролей в захешированном виде;
* проверка хеша пароля при аутентификации;
* защита административных маршрутов;
* проверка существования связанных записей;
* использование транзакций при бронировании;
* защита от повторного бронирования занятого места;
* генерация уникального кода билета.

Пароли не должны храниться в базе данных в открытом виде.

Для создания хеша используется:

```php
Hash::make($password);
```

Для проверки пароля используется:

```php
Hash::check($password, $user->password);
```
